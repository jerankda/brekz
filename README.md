## Overview

brekz. is a desktop application that connects directly to the [OpenRouter](https://openrouter.ai) API. It provides a native, dark-themed chat interface with conversation history, streaming responses, and support for every model OpenRouter exposes -- from Anthropic's Claude to OpenAI's GPT, DeepSeek, Google Gemini, and hundreds more.

Your API key is stored locally. Nothing is proxied or sent to a third-party backend.

## Features

- **SSE streaming** -- Real-time token-by-token responses via Server-Sent Events
- **Conversation history** -- All chats persisted to a local SQLite database
- **Auto-generated titles** -- Conversation titles are generated by `gpt-4o-mini` after the first exchange
- **Curated model selector** -- Quick-access to Anthropic, OpenAI, DeepSeek, Meta, and Google models, with a search field to find any of the hundreds of models OpenRouter provides
- **Markdown rendering** -- Full GitHub-Flavored Markdown (tables, task lists, code blocks, strikethrough)
- **Token and cost tracking** -- Input/output token counts displayed per message
- **Settings panel** -- API key management (save, validate, remove), default model, temperature, and max tokens
- **Collapsible sidebar** -- Conversation list sorted by recency, with delete on hover
- **Error boundary** -- Catches render crashes gracefully with a reload option

## Tech Stack

**Frontend:** React 19, TypeScript, Zustand, Tailwind CSS v4, shadcn/ui (Radix primitives), react-markdown

**Backend:** Tauri v2 (Rust), reqwest, rusqlite, tokio, eventsource-stream

**Storage:** SQLite via rusqlite -- conversations and messages stored locally

## Getting Started

### Prerequisites

- Node.js 20+
- Rust toolchain (install via [rustup](https://rustup.rs))
- Platform-specific Tauri dependencies ([guide](https://v2.tauri.app/start/prerequisites/))

### Development

```bash
git clone https://github.com/jerankda/brekz.git
cd brekz
npm install
npm run tauri dev
```

### Build

```bash
npm run tauri build
```

Output bundles are written to `src-tauri/target/release/bundle/`.

## Downloads

Pre-built binaries for macOS (.dmg), Windows (.msi), and Linux (.deb) are available on the [releases page](https://github.com/jerankda/brekz/releases).

## Architecture

The frontend communicates with the Rust backend through Tauri's `invoke` mechanism. Streaming responses work via Tauri events -- Rust reads the SSE stream from OpenRouter and emits `stream-chunk` events as tokens arrive, which the React frontend renders in real time.

```
React (Zustand stores)  <--invoke/listen-->  Rust (Tauri commands)  <--HTTP/SSE-->  OpenRouter API
                                               |
                                            SQLite (local)
```
